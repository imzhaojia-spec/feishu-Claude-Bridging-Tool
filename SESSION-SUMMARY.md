## 一、项目背景和目标

**做什么**：手机飞书发消息 → Windows 电脑上 Claude Code 处理 → 飞书回复。人不在电脑前也能用 Claude。

**最终交付**：两套代码——
- `feishu-bridge`：你个人用的桥接（已加固安全）
- `feishu-bridge-dist`：给别人用的发行版（17 个文件，README 引导，5 分钟装完）

---

## 二、已确定的决策

| 领域 | 决策 | 原因 |
|------|------|------|
| **会话定义** | 一个飞书聊天窗口 = 一个独立 Claude 会话，chat_id 对 session_id，桥接启停不影响 | 上下文存在磁盘（`sessions.jsonl`），桥接关过再开聊天记录不丢 |
| **数据模型** | 追加式 `.jsonl`，不覆盖旧记录 | 同名/改名/cd 操作只追加不删，crash 安全；`/resume` 列表由最新记录决定 |
| **安全模型** | 三层防御：飞书可见范围 + 首条消息自动绑定主人 + 可选手动白名单 | `--dangerously-skip-permissions` 必须保留（去掉就卡死），安全靠"只有主人能触发"保障 |
| **死信** | 不存消息原文，只存 `chat_id + 错误原因 + 重试数` | 防聊天记录泄露到磁盘；内存里仍重试 2 次 |
| **分发策略** | 复制到 `feishu-bridge-dist` 单独改，不动个人版 | 你个人配置不受影响 |
| **跨平台** | JS 代码天然跨平台；`shell:true` 仅 Windows，Unix 用 `shell:false`；启动脚本分 `.bat` / `.sh` | 已实测确认无硬编码 Windows 路径 |
| **命令系统** | `/help` `/cmds` `/sessions` `/resume` `/resume N` `/new` `/name X` `/name` `/cd X` `/cd` 共 9 条 | 全是桥接秒回，不走 Claude，不花钱 |
| **上下文统计** | `(input_tokens + cache_read_input_tokens) / contextWindow` | 实测验证了公式正确 |

## 六、v1.1 安全加固决策（2026-07-06）

经过 skill-vetter 审查 + 两轮第一性原理讨论，最终方案：

**核心思路**：不发明新认证协议，用已有的白名单机制。首次绑定自动发现 open_id → 写入 .env → 用户重启 → 此后每次消息都显式验证。

**为什么不用绑定码**：绑定码是一次性证明，白名单是每次验证。绑定码需要新命令/新状态/新失败模式，白名单 2 行自动写入就解决。减法优先。

**5 项改动详情**：

| # | 功能 | 关键代码 | 安全效果 |
|---|------|---------|---------|
| ① | 绑定窗口 | `BIND_WINDOW_MS = 5 * 60 * 1000` | 启动 5 分钟后关闭自动绑定 |
| ② | 白名单自动写入 | `autoWriteWhitelist()` 追加到 `.env` | 用户重启即永久白名单模式 |
| ③ | 未授权追踪 | `trackUnauthorized()` Map 分级 | 第 3/10 次终端弹醒目标警告 |
| ④ | /cd 阻断 | `isDangerousPath()` 拒绝系统根 | 拒绝 / /etc C:\Windows 等 |
| ⑤ | 安全摘要 | `printSecuritySummary()` 启动打印 | 一眼确认白名单/窗口/绑定状态 |

**安全模型演进**：
- 首次启动：飞书可见范围 → 5min 窗口 → 未授权追踪 → /cd 阻断（4 层）
- 重启后：显式白名单 → 未授权追踪 → /cd 阻断（3 层，每次消息验证，零窗口）

## 七、技术小白注意事项

| 要点 | 记住这句话 |
|------|-----------|
| **改代码必须重启** | 桥接是 `node src/main.js` 一直跑着的进程，改任何文件后要 Ctrl+C 停掉再重新 `node src/main.js` 或双击 `start.bat`，不重启就不生效 |
| **`.env` 是最重要的文件** | 飞书 App ID、App Secret 在这。`.gitignore` 已经防了提交，但私下发给别人会泄露 |
| **飞书路径用 `/` 别用 `:`** | `/cd E:/Claude` 可以，`/cd E:claude` 不行——Windows 下冒号后面必须跟反斜杠或正斜杠 |
| **`/cmds` 是"装好了没"的试纸** | 飞书发 `/cmds`，收到完整命令手册 = 桥接在正常工作；没收到 = 哪里有问题 |
| **电脑睡眠 = 桥接停** | 这不是云服务，是你电脑上的程序。电脑关机/睡眠后飞书消息没人收 |
| **陌生人发消息被静默丢弃** | 自动绑定后，不是你的人发消息——飞书看起来像没发出去，终端日志会记一笔但谁也不知道桥接存在 |
| **`state/` 目录别删** | 删掉 `state/authorized.jsonl` = 换主人；删掉 `state/sessions.jsonl` = 丢失所有会话映射 |
