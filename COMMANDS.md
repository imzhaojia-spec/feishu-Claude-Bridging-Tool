# 飞书桥接 — 命令速查表

> 在飞书机器人的聊天框里直接输入命令发送即可。所有命令秒回（不走 Claude，不产生费用）。
> 直接发消息（不以 `/` 开头的内容）则转发给 Claude 处理。

## 全部命令一览

| 命令 | 示例 | 作用 |
|------|------|------|
| `/help` | `/help` | 列出所有可用命令 |
| `/sessions` | `/sessions` | 列出所有历史会话（名称、短ID、最后活跃时间） |
| `/resume` | `/resume` | 列出所有可切换的会话（每个有序号、短ID、名称、工作目录、最后活跃时间） |
| `/resume N` | `/resume 3` | 切换到第 3 号会话。下次发消息时自动创建分叉副本（原会话不受影响，多个窗口可同时用同一会话的副本） |
| `/new` | `/new` | 在当前窗口开启新会话，清空上下文。旧会话仍在 `/resume` 列表里可回溯 |
| `/name X` | `/name 重构飞书桥接` | 给当前会话起个名字（最多 50 字）。起名后 `/resume` 列表里好认 |
| `/name` | `/name` | 显示当前会话的名称 |
| `/cd X` | `/cd /your/project` | 切换当前会话的工作目录。后续发消息时 Claude 在 X 目录下运行，能读到那个目录的 CLAUDE.md、skills 等配置。**`/cd` 不切会话，只换工位——`/cd` = 换个目录接着聊，上下文还在；`/resume N` = 换个会话重新开始。** |
| `/cd` | `/cd` | 显示当前会话的工作目录（Claude 在哪个文件夹里跑） |

## 回复统计

每条 Claude 回复末尾自动附加一行上下文统计：

```
━━━━━━━━━━
上下文 134K/1000K (13.4%) · $0.09 · 24.2s · 第5轮
```

| 字段 | 含义 |
|------|------|
| 上下文 | 当前对话占用上下文窗口的百分比。接近 100% 时该 `/new` 了 |
| $ | 本轮对话的 API 费用（美元） |
| 秒数 | Claude 处理本轮用了多少秒 |
| 轮次 | 当前会话总共进行到第几轮对话 |

## 典型工作流

```
# 场景：在飞书上开始一个新项目
/cd /your/project           # 切到项目目录
/name 我的项目讨论           # 起个名字方便以后找
发消息：帮我看看这个项目的架构  # Claude 开始工作

# 场景：想切回之前的对话
/resume                    # 看有哪些会话
/resume 2                  # 切到第 2 号会话
发消息：继续上次的话题        # Claude 从上次断点继续

# 场景：上下文快满了
/help                      # 看统计行，上下文 85%
/new                       # 开新会话清空上下文
```
